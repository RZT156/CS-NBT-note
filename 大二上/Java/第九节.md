# 集合
==存放在java.util==
**数组缺点**
1. 长度不可变；删除、插入效率低；实际元素数量不确定
2. 存储：有序、可重复，无法满足无序、不可重复

**集合 & 数组**
1. 集合长度可变
2. 集合存储==引用数据类型==（对象），存基本类型要用包装类（大写）
==`ArrayList<Integer>list = new ArrayList<>();`==
这个是存都是Integer类型的数据，如果自定义类修改Interger即可
`ArrayList list = mew ArrayList();`
可以存任何但是被淘汰别学，不安全（add就直接用，要get给别的变量需要（）强制类型转换）
3. 不同集合底层数据结构不同

**集合类**
list：有序、可重复，有索引
	遍历：for循环；迭代器
set：无序、不重复，无索引
ArrayList：底层基于数组，大小可变
	遍历：for循环；增强for==不适用于int==；迭代器；JDK1.8之后lambda表达式······
	自带`ArrayList变量名.add(代添加变量);remove()方法`
	remove方法使用同add
**Collection**是根接口（就像Object和类的关系）
- 迭代器
Iterator iterator()返回collection上进行迭代的Iterator

# 泛型
**泛型**
相当于标签<>，在具体使用的时候才会确定类型
可以确定在集合中存放的数据类型
只能为引用数据类型（对象）
**泛型标识**
- E - 元素
- T - Java类
- K - 键
- V - 值
- N - 数值类型
`class (类名)<泛型标识,泛型标识,···>{`
	`private T 变量名;`
`}`
Generic 类；Generic\<T>泛型
**实例化对象时，不指定泛型相当于Object类
实例化对象时，指定泛型（推荐）**
