# 3.1 概论
### 存储系统层次结构
分层次原因：
- 速度快的容量小，贵
- 便宜的速度慢容量大
分层次目的：
解决==存储容量、价格和存储速度==之间的矛盾
对存储器要求==容量大、速度快、成本低==
**三级存储系统：cache+主存+辅存**
内存=cache+主存
**两层存储系统：cache-主存层；主存-辅存层（均为速度接近前者，容量和价格接近后者）**
cache-主存层由硬件自动完成，对所有程序员透明；主存-辅存层由硬件和操作系统共同完成，对应用程序员透明。
![[存储系统层次结构.jpg]]
### 存储器分类
**RAM(random access memory)**
掉电丢失数据，速度快
**ROM(read only memory)**
掉电不丢失数据，只读，速度慢
ROM（出厂前设置好不可改变）->PROM（出厂后可改一次）->
EPROM（紫外光可擦）->EEPROM（电可擦）->Flash（闪存）
### 性能指标
**存储容量**
存储字数（存储单元数）×存储字长（每单位比特数）
**存取时间**：一次读/写操作发出到完成
**存储周期**：“CPU连续两次访问存储器的最小间隔时间”
存取时间+线路恢复时间
**存取器带宽**：单位时间存取信息量，单位B/S或者bit/s
总线W/存取周期
# 3.2 静态随机存取存储器SRAM
### 结构
static RAM：存储位元是多个门电路组成的锁存器，一般用于高速缓冲存储器
存储单元 = $2^{地址线数}$ 个
数据位数 = **数据线数**
存储容量 = 存储字数（存储单元）×存储字长（数据位数）
**控制线**：控制读/写
### 存储器容量扩充
 要求存储器容量VS给定芯片容量 
1. 位拓展 字数一致，位数较短
方法：==数据线分开连==，地址线、控制线共用
描述：将x片【给定芯片】进行位扩展，三组信号线中，...共用，数据线每 y($y=总数据线数/x$) 根分开连接到x片【给定芯片】，构成【要求芯片】
2. 字拓展 字数较少，位数一致
方法：地址总线产生片选信息，多片芯片==分时工作==。三线共用
高位地址线（->片选）+普通地址线=总地址线=要求地址线（共用）
数据线 = 要求数据线
控制线 = 要求控制线
3. 字位拓展 先位拓展，再字拓展存储单元
描述：先位扩展，用x片【给定芯片】构成【给定字×要求位】芯片组，形成完整的【x位bit】存储单元。
再用y组【给定字×要求位】的芯片组进行字拓展，构成【要求芯片】
# 3.3 动态随机存取存储器DRAM
### 结构
存储位元：**MOS管+电容器**，主要用于主机内存

**增加行、列地址锁存器**，行列地址分时传送。
存储容量一半的地址线

**增加刷新计数器**，读出后再生
刷新：对DRAM定期进行全部重写过程。
原因：电容泄漏引起DRAM信息衰减
### 刷新方法
- 集中式
在最大刷新间隔时间内，找一段时间进行刷新。读写操作不受刷新工作影响
缺点：不可进行读写操作，存在CPU访问“死时间“
- 分散式
在每个读/写周期后插入刷新周期，无”死时间”。
缺点：加长系统存取周期，降低整机速度
- 异步刷新

# 3.6 高速缓冲存储器cache
### 基本概念
cache相对其他内存速度更快，内存更小。提升访问速度，以使存取速度和CPU运算速度相匹配
原理：
CPU读取时（CPU只能直接访问内存），分别向内存和cache发送访问请求。cache命中则直接传主存访问失败；cache未命中则从主存发送，同时复制给cache一份
### 命中率
cache命中率$h$ = cache存取次数$N_c$ / (cache存取次数$N_c$+主存存取次数$N_m$)
平均访问时间$t_a$ = $h$ × cache访问时间$t_c$ + $(1-h)$ × 主存访问时间$t_m$
访问效率$e$ = $t_c$/$t_a$
### 映射
1. 全相联：主存数据块可复制到cache任意一行
2. 直接相联：主存数据块只能到固定某行
3. 组相联：复制到固定某组的任意行
==LFU替换算法==：替换的是使用次数最少的
